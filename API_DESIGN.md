# APIのデザイン

全国まとめてデータを入手しても、タイル分割まで含めて5秒で終わるので、airpollutionwatch(APW)からは日本全体を一括して読みこむ。

補間アルゴリズムは選べるようにする。

補間した結果は(APIで)キャッシュする。細かさは地理院タイル12で十分だと思うが、14にすると拡大してもなめらかになる。処理時間としては、14で全国でも十分可能。

利用者がどのような形でデータを取得するかは未定。相談して決める。

一般的な利用形態としては、ある県に含まれる地理院タイル12の指定された複数のタイル点における測定値x,y,zのデータを、from時からto時までほしい、みたいな感じか。局とか県とかはもはや要らない。

→ 詳細な API 仕様は [docs/grid_api_spec.md](docs/grid_api_spec.md) を参照。

## 検討課題

- APWでのデータ収集は正時から15分ぐらいまでかかる。関東に限定すれば、取得は正時から数秒で完了するので、すぐに内挿にとりかかれるが、全国一括だと全部そろうのにけっこう時間がかかる上に、下手をすると一部の県で欠測があっていつまでたっても最新データをとれない可能性もある。APWはその場合、古いデータも混ざった状態で、タイムスタンプつきのデータを提供する。こちらのAPIがデータを提供する時には、APWからいつ取得したデータで内挿したのかを示すとともに、定期的(5分に一回程度)にAPW APIをつついて、一部データの更新があった場合には内挿を再構築するなどの対応が必要。
